## 🖐️ 手动触发更新

### 方式 1: GitHub 网页
1. 访问：https://github.com/ZW72212342/authorities-directory/actions/workflows/update-news.yml
2. 点击 "Run workflow" 按钮
3. 选择要更新的机构（或留空更新全部）
4. 点击 "Run workflow"

### 方式 2: GitHub CLI
```bash
# 更新全部机构
gh workflow run update-news.yml

# 更新特定机构
gh workflow run update-news.yml --field org=miit
```

可用机构代码：
- `miit` - 工信部
- `ndrc` - 发改委
- `most` - 科技部
- `cac` - 网信办

## 📊 查看更新日志

```bash
# 查看最近的更新
gh run list --workflow=update-news.yml --limit 10
```

## 🔧 自定义配置

### 修改抓取 URL
编辑 `.github/scripts/fetch_news.py` 中的 `ORG_CONFIG`：

```python
ORG_CONFIG = {
    'miit': {
        'name': '工业和信息化部',
        'url': 'www.miit.gov.cn/xwfb/zxzc/index.html',
        'base': 'http://www.miit.gov.cn',
        'keywords': ['软件', '数据', '人工智能', '智能制造', '信息系统']
    },
    # ... 其他机构
}
```

### 修改更新频率
编辑 `.github/workflows/update-news.yml` 中的 `schedule`：

```yaml
on:
  schedule:
    # cron 表达式 (UTC 时间)
    - cron: '0 0,8,16 * * *'  # 每天 3 次
```

### 添加新机构
在 `ORG_CONFIG` 中添加新条目：

```python
'new_org': {
    'name': '新机构名称',
    'url': '网址路径',
    'base': '基础 URL',
    'keywords': ['关键词 1', '关键词 2']
}
```

### 常见问题

**Q: 为什么某些机构没有更新？**
A: 可能是网站暂时无法访问或没有符合关键词的新政策。系统会自动使用上次的成功数据。

**Q: 如何验证更新是否成功？**
A: 
1. 检查 GitHub Actions 运行状态（绿色✓表示成功）
2. 查看 HTML 文件中的 `LATEST_NEWS_DATA` 是否更新
3. 刷新 GitHub Pages 页面查看最新内容

**Q: 可以实时更新吗？**
A: GitHub Actions 最短执行间隔为 5 分钟。如需更高频率，建议：
- 使用云服务（如 AWS Lambda + EventBridge）
- 使用本地服务器定时任务
- 使用第三方监控服务（如 cron-job.org）
